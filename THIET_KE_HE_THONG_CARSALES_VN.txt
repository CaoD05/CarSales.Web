2.1 KIẾN TRÚC TỔNG QUÁT

Hệ thống CarSales.Web được thiết kế theo kiến trúc phân lớp (Layered Architecture):

    ┌──────────────────────────────────────────────────────┐
    │     LỚP TRÌNH BÀY (Presentation Layer)               │
    │  - Giao diện Razor Pages                             │
    │  - API Controllers                                   │
    │  - Các thành phần UI                                 │
    ├──────────────────────────────────────────────────────┤
    │     LỚP LOGIC KINH DOANH (Business Logic Layer)      │
    │  - Controllers                                       │
    │  - Xử lý yêu cầu người dùng                          │
    │  - Các API Endpoints                                 │
    ├──────────────────────────────────────────────────────┤
    │     LỚP TRUY CẬP DỮ LIỆU (Data Access Layer)         │
    │  - Entity Framework Core                             │
    │  - DbContext                                         │
    │  - Các Migration                                     │
    ├──────────────────────────────────────────────────────┤
    │     LỚP CƠ SỞ DỮ LIỆU (Database Layer)               │
    │  - SQL Server                                        │
    └──────────────────────────────────────────────────────┘

3.1 HỆ THỐNG XÁC THỰC & QUẢN LÝ NGƯỜI DÙNG

Các Thành Phần:
  • AccountController.cs
  • AuthApiController.cs
  • RoleAuthorizeAttribute.cs
  • User.cs, Role.cs

Các Chức Năng:
  ✓ Đăng Ký Tài Khoản
    - Xác nhận email hợp lệ
    - Mã hóa mật khẩu
    - Gán vai trò (Admin, Nhân viên, Khách hàng)
    - Tạo hồ sơ người dùng

  ✓ Đăng Nhập
    - Xác thực qua email
    - Quản lý session (30 phút hết hạn)
    - Điều hướng theo vai trò

  ✓ Quản Lý Hồ Sơ
    - Cập nhật thông tin cá nhân
    - Tải lên ảnh đại diện
    - Cập nhật liên hệ

  ✓ Phân Quyền Dựa Trên Vai Trò (RBAC)
    - 3 loại vai trò: Admin, Nhân viên, Khách hàng
    - Kiểm tra phân quyền tự động
    - Cấu trúc vai trò: Admin > Nhân viên > Khách hàng

Quy Trình:
  1. Người dùng đăng ký với email và mật khẩu
  2. Mật khẩu được mã hóa trước lưu trữ
  3. Đăng nhập bằng email/mật khẩu
  4. Session lưu trữ UserId và Role
  5. Mỗi yêu cầu được kiểm tra phân quyền
  6. Session hết hạn sau 30 phút không hoạt động

Các Chức Năng:

    - Hiển thị tất cả xe có sẵn
    - Lọc theo loại xe
    - Sắp xếp theo nổi bật & ngày tạo
    - Hỗ trợ phân trang
    - Hiển thị ảnh xe

  ✓ Tìm Kiếm & Khám Phá
    - Lọc theo hãng xe
    - Lọc theo loại xe
    - Lọc theo khoảng giá
    - Lọc theo năm sản xuất
    - Lọc theo loại nhiên liệu
    - Tìm kiếm theo tên xe

  ✓ Trang Chi Tiết Xe
    - Thông số kỹ thuật đầy đủ
    - Nhiều hình ảnh xe
    - Thông tin giá và tình trạng
    - Gợi ý xe liên quan

  ✓ Thông Tin Xe Bao Gồm
    - Tên xe, Hãng, Loại xe
    - Giá bán, Năm sản xuất
    - Số km đã chạy
    - Loại nhiên liệu, Hộp số
    - Số ghế, Màu sắc
    - Động cơ, Xuất xứ
    - Mô tả chi tiết

Quy Trình:
  1. Khách hàng truy cập Trang Chủ
  2. Chọn loại xe để lọc
  3. Xem danh sách xe được sắp xếp
  4. Nhấp vào xe để xem chi tiết
  5. Xem các thông số kỹ thuật và ảnh
  6. Có thể thêm vào yêu thích hoặc mua

3.8 CÁC ENDPOINTS API

1. BrandsApiController.cs
   GET    /api/brands            - Lấy tất cả hãng xe
   POST   /api/brands            - Tạo hãng xe mới
   GET    /api/brands/{id}       - Lấy hãng xe theo ID
   PUT    /api/brands/{id}       - Cập nhật hãng xe
   DELETE /api/brands/{id}       - Xóa hãng xe

2. CarsApiController.cs
   GET    /api/cars              - Lấy tất cả xe (có lọc)
   POST   /api/cars              - Tạo xe mới
   GET    /api/cars/{id}         - Lấy xe theo ID
   PUT    /api/cars/{id}         - Cập nhật xe
   DELETE /api/cars/{id}         - Xóa xe

3. AuthApiController.cs
   POST   /api/auth/login        - Đăng nhập người dùng
   POST   /api/auth/register     - Đăng ký người dùng
   POST   /api/auth/logout       - Đăng xuất

4. PurchaseRequestsApiController.cs
   GET    /api/purchaserequests  - Lấy yêu cầu mua
   POST   /api/purchaserequests  - Tạo yêu cầu mới
   GET    /api/purchaserequests/{id} - Lấy theo ID
   PUT    /api/purchaserequests/{id} - Cập nhật trạng thái

5. DepositsApiController.cs
   GET    /api/deposits          - Lấy danh sách tiền đặt cọc
   POST   /api/deposits          - Ghi nhận tiền mới
   GET    /api/deposits/{id}     - Lấy theo ID

6.1 XÁC THỰC & PHÂN QUYỀN

  • Xác Thực Dựa Trên Session
    - Hết hạn không hoạt động 30 phút
    - Cookies HttpOnly (ngăn truy cập JavaScript)
    - Cờ Secure cho truyền HTTPS

  • Bảo Mật Mật Khẩu
    - Mã hóa mật khẩu bằng PasswordHelper
    - Không bao giờ lưu ở định dạng plain text
    - Muối được áp dụng khi mã hóa

  • Phân Quyền Dựa Trên Vai Trò
    - Custom RoleAuthorizeAttribute
    - Xác nhận vai trò trên mỗi yêu cầu bảo vệ
    - Ba cấp độ vai trò: Admin, Nhân viên, Khách hàng

  • Xác Nhận Session
    - UserId và Role được lưu trong session
    - Được kiểm tra trước khi cấp quyền truy cập
    - Sessions không hợp lệ điều hướng đến đăng nhập


6.2 BẢO VỆ DỮ LIỆU

  • Ràng Buộc Khóa Ngoài
    - Ngăn chặn dữ liệu mồ côi
    - Hành vi OnDelete: Restrict (ngăn chặn mất dữ liệu)

  • Xác Nhận Đầu Vào
    - Xác nhận trường bắt buộc
    - Xác nhận định dạng email
    - Xác nhận số điện thoại
    - Ràng buộc độ dài chuỗi

  • Ngăn Chặn SQL Injection
    - Truy vấn tham số hóa Entity Framework Core
    - Không xây dựng SQL động


6.3 BẢO MẬT API

  • Headers CORS
    - Kiểm soát yêu cầu cross-origin

  • Tài Liệu Swagger
    - Chỉ có sẵn ở chế độ phát triển

  • Quản Lý Session Token
    - Xác nhận ID session
    - Bảo vệ CSRF thông qua trạng thái session

  • Phân Trang
    - Xử lý tập kết quả lớn bằng phân trang
    - Giới hạn kết quả truy vấn cơ sở dữ liệu

  • Caching (Bộ Nhớ Đệm)
    - Distributed Memory Cache có sẵn
    - Có thể bộ nhớ đệm dữ liệu được truy cập thường xuyên

  • Caching Session
    - Dữ liệu session người dùng được lưu trữ trong bộ nhớ
    - Giảm chi phí chung xác thực


8.2 XEM XÉT KHẢ NĂNG MỞ RỘNG

  • Pooling Kết Nối Cơ Sở Dữ Liệu
    - Pooling kết nối SQL Server
